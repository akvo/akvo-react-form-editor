## Confirmed JSON Shapes (from example.json)

### geo (existing, Q2 + Q52)
```json
{ "type": "geo", "center": { "lat": 9.145, "lng": 40.4897 } }
{ "type": "geo", "center": [-18.1236015, 178.3805867] }
```
Two formats — normalize to array in editor.

### geotrace (Q97) / geoshape (Q98)
```json
{ "type": "geotrace", "center": [-18.1236015, 178.3805867] }
{ "type": "geoshape", "center": [-18.1236015, 178.3805867] }
```

### signature (Q78)
```json
{ "type": "signature" }
```
No special config.

### attachment (Q75–77) — KEY: allowedFileTypes inside `rule`
```json
{ "type": "attachment", "rule": { "allowedFileTypes": ["jpeg","jpg","png"] } }
{ "type": "attachment", "rule": { "allowedFileTypes": [] }, "api": { "endpoint": "...", "response_key": "location" } }
{ "type": "attachment", "rule": { "allowedFileTypes": ["pdf","doc","docx","odt"] }, "api": { "endpoint": "...", "response_key": "location" } }
```
- `allowedFileTypes` is **inside `rule`**, NOT top-level
- Values are file **extensions** (not MIME types): jpeg, jpg, png, pdf, doc, docx, odt
- `api.response_key` is the key in the upload response containing the file URL
- `api.query_params` is auto-generated by renderer — **not configured in editor**

### entity (Q67–68) — KEY: it's cascade + extra
```json
{
  "type": "cascade",
  "api": { "endpoint": "https://..." },
  "extra": { "type": "entity", "name": "School", "parentId": 5 }
}
```
Entity is NOT a separate question type. It is cascade with an `extra` object.

### partialRequired (Q28)
```json
{ "type": "cascade", "partialRequired": true, "api": { ... } }
```
Top-level question property.

### tree (Q24)
```json
{ "type": "tree", "option": "administration", "checkStrategy": "children", "expandAll": true }
```
Both top-level question properties.

### leading_question / show_repeat_in_question_level (Group 12)
```json
{
  "id": 12,
  "repeatable": true,
  "leading_question": 88,
  "show_repeat_in_question_level": true,
  "question": [ ... ]
}
```
Group-level properties. `leading_question` = question ID in a **different group**.

### is_repeat_identifier (Q89)
```json
{ "id": 89, "type": "input", "is_repeat_identifier": true }
```
Question-level boolean. Inside a repeatable group.

### dependency_rule (Q94 OR, Q96 AND)
```json
{ "dependency_rule": "OR", "dependency": [ {"id":91,"options":["No"]}, {"id":93,"options":["Yes"]} ] }
{ "dependency_rule": "AND", "dependency": [ {"id":91,"options":["Yes"]}, {"id":95,"options":["Yes"]} ] }
```
Question-level string, always present alongside 2+ dependency entries.
